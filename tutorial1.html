Difference between Structured Oriented and Object Oriented Programming.

Procedure-Oriented / Structured Programming:

  • Structure Oriented Programming is also known as Procedure Oriented Programming.
  • It is basically a sequence of instructions that need to be done for solving a problem.
  • For doing so, number of function need to be written for completing the task.
  • Function have their sequence and sequence of callings.
  • Each function contains list of instructions inorder to excuting them in proper sequence.
  • Each function had it data and local variables.many important data items are placed as global. so, that they may accessed by all the functions.
  • Global data are more reluctant to change by a function.
  • In large program it is quite difficult to identify global variables. This urgently needs a revision in external data structure.

Drawbacks of Structured Programming:

  • Emphasis on developing Algorithm.
  • Large program divided into smaller parts known as Function.
  • Function use to share global data.
  • Data moves openly from one function to another.
  • It employs top down approach in program design.
  • Function transform data into different form.

Object Oriented Programming
  • Object Oriented Programming treat data as critical element in the program development and donot allow it's free flow.
  • It used to tie data and function together and prevent data from accidental modification from outside function.
  • It allow decomposintion of problem into a entity called an Object.
  • Once these entities are defined then data member and member function are difined to work upon.
  • This is an approach that's provide a way of modularizing by creating memory area for both data menber and member function in the form of template. so as to provide copy of such modules on demand.



Basic concept of Object Oriented Programming:

  • Class:
  • Class contains data and member function to work on that data.
  • It is a user defined type.
  • Class is a collection of object of similar type.
    •                         /*for example:*/

                        class human{  /*here, human is the class...*/
                            private:
                                char physicalAppereance;

  • Object:
  • Person, Place, Things are entities and these entities has features and characterstics, these entites are called an Object.
    • in the above example, if human is a class then object can be male and female...

  • Abstraction:
  • Abstraction refers to representing essential details or features without including background details or functionality.
    • may be everybody drives a car or bike or any vehicle.

  • Encaptulation:
  • Wrapping up of data and member function into a single unit is known as Encaptulation.

  • Inheritance:
  • Inheritance is the feature where property of one class is acquired by another class.
  • Inheritance helps in implementing the concept of code reusability.

  • Polymorphism:
  • Polymorphism is a concept where a function response differently on the basis of input.
  • Here input is the parameter, that is used to pass.
  • Polymorphism cames from Greek where "Poly" means "many" and "Morphism" means "shape".
  • Polymorphism helps in performing different task or operation by overloading a function and that function will be called with same name but with different parameters and operations.
