basic concept of OOP

  • Object-Oriented Programming (OOP) is a programming paradigm based on the concept of objects.
  • These objects are instances of classes, which define their properties (attributes) and behaviors (methods).
  • OOP helps in organizing code in a more modular, reusable, and scalable way.

key concept of OOP:

1. Class and Object

  • A Class is an blueprint for creating objects.
  • An Object is an istance of Class. it is an run time entities.
Example
#include <iostream>
  using namespace std;
                  
  class Car {
  public:
      string brand;
      string model;
                  
      void display() {
          cout << "Car: " << brand << " " << model << endl;
      }
  };
                  
  int main() {
      Car car1;
      car1.brand = "Toyota";
      car1.model = "Corolla";
      car1.display();  // Output: Car: Toyota Corolla
                  
      return 0;
  }

2. Encaptulation (Data Hiding)

  • Encapsulation protects data by restricting direct access to class members.
  • Private members cannot be accessed directly outside the class.
Example
#include <iostream>
  using namespace std;
                  
  class BankAccount {
  private:
      double balance;
                  
  public:
      BankAccount(double bal) { balance = bal; }
                  
      void deposit(double amount) { balance += amount; }
                  
      double getBalance() { return balance; }
  };
                  
  int main() {
      BankAccount account(1000);
      account.deposit(500);
      cout << "Balance: " << account.getBalance() << endl;  // Output: Balance: 1500
      return 0;
  }

3. Inheritance

  • Inheritance allows a class to acquire properties and behavior from another class.
Example
#include <iostream>
  using namespace std;
                
  class Vehicle {
  public:
      string brand = "Ford";
                
      void honk() { cout << "Beep Beep!" << endl; }
  };
                
  class Car : public Vehicle {
  public:
      string model = "Mustang";
  };
                
  int main() {
      Car myCar;
      cout << myCar.brand << " " << myCar.model << endl;  // Output: Ford Mustang
      myCar.honk();  // Output: Beep Beep!
      return 0;
  }

4. Polymorphism

  • Polymorphism allows functions to behave differently based on the object type.
  • Achieved through function overriding or function overloading.
Function Overriding (Using Virtual Functions)
#include <iostream>
  using namespace std;
                
  class Animal {
  public:
      virtual void speak() { cout << "Animal speaks" << endl; }
  };
                
  class Dog : public Animal {
  public:
      void speak() override { cout << "Woof!" << endl; }
  };
                
  int main() {
      Animal* a;
      Dog d;
      a = &d;
        a->speak();  // Output: Woof!
        return 0;
    }

Function Overloading
              #include <iostream>
                using namespace std;
                
                class Math {
                public:
                    int add(int a, int b) { return a + b; }
                    double add(double a, double b) { return a + b; }
                };
                
                int main() {
                    Math m;
                    cout << m.add(2, 3) << endl;    // Output: 5
                    cout << m.add(2.5, 3.5) << endl; // Output: 6.0
                    return 0;
                }

5.Abstraction

  • Abstraction hides implementation details and only exposes essential functions.
  • Achieved using pure virtual functions in an abstract base class.
Example
#include <iostream<
  using namespace std;
                  
  class Shape {
  public:
      virtual double area() = 0; // Pure virtual function (makes the class abstract)
  };
                  
  class Circle : public Shape {
  private:
      double radius;
                  
  public:
      Circle(double r) { radius = r; }
                  
      double area() override { return 3.14 * radius * radius; }
  };
                  
  int main() {
      Circle c(5);
      cout << "Area: " << c.area() << endl;  // Output: Area: 78.5
      return 0;
  }

benifits of object-oriented programming (OOP)


1. code reusabilit (inheritance)

  • OOP allows reuse of existing code through inheritance, reducing redundancy.
  • Example: A Car class can inherit properties from a Vehicle class instead of redefining them.

2. encaptulation (data hiding)

  • Protects data by restricting access to certain parts of an object.
  • Example: Private variables in a BankAccount class prevent unauthorized access.

3. Modularity (code orginazation)


  • OOP structures code into independent objects, making it more organized and modular.
  • This helps in team collaboration since different teams can work on separate objects.

4. Scalability and Maintainability


  • Large applications become easier to manage due to a well-structured object hierarchy.
  • Example: Updating a single class affects all objects derived from it.

5. Flexibility (Polymorphism)


  • OOP allows functions to have different behaviors based on the object that calls them.
  • Example: A draw() function can work for both Circle and Rectangle classes without rewriting code.

6. Abstraction (Simplified Complexity)


  • Hides complex implementation details from the user, showing only the necessary parts.
  • Example: A Car class provides a startEngine() method without exposing internal mechanisms.

Tokens and identifiers:


Tokens:

  • Keywords: Reserved words like int, return, class, if, while, etc.
  • Identifiers: Names for variables, functions, arrays, etc. (e.g., myVar, sum, calculate()).
  • Literals: Constant values like 10, 3.14, 'A', "Hello".
  • Operators: Symbols that perform operations, like +, -, *, /, ++, ==.
  • Punctuators (Separators)Symbols like {}, ;, ,, (), [].
  • Comments: Non-executable text written with // (single-line) or /*...*/ (multi-line).

Identifiers:

  • Identifiers are user-defined names used for variables, functions, arrays, and other elements.

  • Rules for Naming Identifiers:
  • Can contain letters (A-Z, a-z), digits (0-9), and underscore (_).
  • Must start with a letter or an underscore (not a digit).
  • Cannot be a keyword (int, float, while, etc.).
  • Case-sensitive (sum and Sum are different).
  • No spaces or special characters (@, $, %).
Example
int age;  // 'age' is an identifier
float price;  // 'price' is an identifier
void calculateTotal() { } // 'calculateTotal' is an identifier

  • 1. Letters (Alphabets)
  • Uppercase: A-Z
  • Lowercase: a-z

  • 2. Digits
  • 0-9 (Used in numbers and identifiers)

  • 3. Special Characters (Symbols)

  • 4. Whitespace Characters
  • Space ( ), Tab (\t), Newline (\n), Carriage Return (\r), Form Feed (\f)

  • 5. Escape Sequences (Special characters using \)
  • \n (Newline), \t (Tab), \b (Backspace), \\ (Backslash), \' (Single quote), \" (Double quote)

Symbols in C++

  • Symbols in C++ are special characters used for various purposes, such as arithmetic, logical operations, and program structure.

  • 1. Arithmetic Operators
  • + (Addition), - (Subtraction), * (Multiplication), / (Division), % (Modulus)

  • 2. Relational Operators
  • == (Equal to), != (Not equal to), > (Greater than), < (Less than), >= (Greater than or equal to), <= (Less than or equal to)

  • 3. Logical Operators
  • && (Logical AND), || (Logical OR), ! (Logical NOT)

  • 4. Bitwise Operators
  • & (AND), | (OR), ^ (XOR), ~ (NOT), << (Left shift), >> (Right shift)

  • 5. Assignment Operators
  • = (Assignment), +=, -=, *=, /=, %=, &=, |=, ^=, <<=, >>=

  • 6. Increment & Decrement Operators
  • ++ (Increment), -- (Decrement)

  • 7. Punctuation & Separators
  • {} (Braces for blocks), () (Parentheses for functions), [] (Brackets for arrays), ; (Statement terminator), , (Separator)

  • 8. Other Symbols
  • ? : (Ternary operator), -> (Pointer operator), :: (Scope resolution operator), sizeof (Size of data type)

Variables in C++

  • A variable is a name given to a memory location that is used to store data.
  • The value of a variable can be changed during the execution of the program.

  • Syntax:
  • int age = 20;
  • Here, age is a variable of type int (integer) and it stores the value 20.

Constants in C++

  • A constant is a value that cannot be changed during the execution of the program.
  • In C++, constants are declared using the const keyword.

  • Syntax:
  • const data_type constant_name = value;

  • Example
  • const float PI = 3.14;
  • Here, PI is a constant of type float, and its value is fixed as 3.14.

Dynamic Initialization of Variables

  • It means initializing variables at runtime using expressions or values.
  • Initialization happens using the result of a calculation or function.
  • Useful when the initial value is not known at compile time.

  • Examples:
  • int a = 5, b = 10;
  • int sum = a + b; // dynamically initialized

Reference Variables

  • A reference is an alias (another name) for an existing variable.
  • Declared using the & symbol.
  • Any changes made using the reference also affect the original variable.

  • Example:
  • int x = 10;
  • int &ref = x; // ref is a reference to x
  • ref = 20; // x is now 20

Basic Data Types in C++

  • These are the fundamental types used to declare variables.
  • Common basic data types:
  • int - for integers (e.g., 5, -3)
  • float - for decimal numbers (e.g., 3.14)
  • double - for larger decimal numbers
  • char - for characters (e.g., 'A')
  • bool - for Boolean values (true or false)

  • Example:
  • int age = 18;
  • float pi = 3.14;
  • char grade = 'A';
  • bool isPass = true;
